# Zx50 Bus CPLD Core

The Zx50 Bus features a distributed, CPLD-driven Memory Management Unit (MMU) designed for the Z80 processor. It allows multiple discrete 1MB memory cards to share the Z80 backplane seamlessly, dynamically translating logical CPU addresses to physical SRAM addresses on the fly.

For deep technical details on the paging protocol, port mapping, and the Shadow Bus DMA architecture, please see [Zx50 Memory.md](Zx50\ Memory.md).

## Project Organization & Toolchain

This project has migrated away from legacy proprietary IDEs (Quartus/ISE) to a modern, `Makefile`-driven open-source synthesis flow using Yosys and the Atmel Fitter.

```text
Zx50Bus/
├── Makefile        # Master build script (Simulation, Synthesis, and Flashing)
├── build/          # Compiled iverilog simulation binaries
├── docs/           # Datasheets and architecture notes
├── sim/            # Verilog testbenches (e.g., zx50_dual_mem_card_tb.v)
├── src/            # Pure Verilog RTL source files (zx50_cpld_nodma.v)
├── syn/            # Synthesis artifacts generated by Yosys/Fitter
│   ├── yosys.log               # Yosys dry-run and linting logs
│   ├── zx50_cpld_nodma.edif    # Intermediate netlist
│   ├── zx50_cpld_nodma.fit     # Atmel Fitter routing report (Check this for utilization!)
│   ├── zx50_cpld_nodma.jed     # The raw Atmel JEDEC binary
│   └── zx50_cpld_nodma.svf     # The JTAG SVF file ready for OpenOCD flashing
└── waves/          # VCD waveform dumps for GTKWave
```

## Build Commands

* **`make`**: Compiles and runs all `iverilog` testbenches.
* **`make yosys-check`**: Runs a Yosys dry-run to check for syntax errors and estimate gate counts without invoking the legacy Atmel fitter.
* **`make syn_nodma`**: Runs the full synthesis flow (Yosys -> Atmel Fitter -> SVF Converter) for the NoDMA fallback architecture.
* **`make flash_nodma`**: Flashes the resulting `.svf` file to the CPLD via OpenOCD.
